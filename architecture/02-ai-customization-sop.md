# SOP 02: AI Roadmap Customization

## 1. Goal
Take the full JSON roadmap generated by `ingest_roadmap.py` and trim/personalize it based on standard `User Input Schema` using an LLM.

## 2. Inputs
- `User Input Schema` JSON string.
- Extracted Roadmap JSON string (from `.tmp/`).

## 3. Tool Logic (Python)
The deterministic python tool (`tools/ai_roadmap_trimmer.py`) will:
1. Parse inputs and ensure they meet the defined JSON Schemas.
2. Construct a strict prompt requiring the LLM to output ONLY a JSON payload adhering to the `Roadmap Schema`.
3. The prompt restricts the LLM from adding hallucinated resources, referencing ONLY vetted resources or general topic placeholders for Phase 1.
4. Execute the LLM call probabilistically, but wrap the result in a deterministic JSON schema validator.
5. If the LLM generates invalid JSON, fallback to a deterministic naive subset of the original roadmap.
6. Write the final customized JSON to `.tmp/{role}_customized.json`.

## 4. Edge Cases
- **LLM Rate Limits/Failures**: Return the unmodified ingested roadmap as a graceful fallback.
- **Malformed JSON Response**: Use a regex/JSON parser that strips markdown ticks (```json ... ```) before parsing.

## 5. Usage
```bash
python tools/ai_roadmap_trimmer.py --role frontend --user_goals '{"experience_level": "beginner", "time_commitment": 10}'
```
